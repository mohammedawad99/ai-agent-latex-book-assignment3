# LaTeX Project (`latex_project/`)

This directory holds the LaTeX sources for the article *From PoC to
Production: A CrewAI Multi-Agent Pipeline for Generating a LaTeX Book*
(group MaRs-777; Mohamed Awad, Rawey Sleiman).

**Status: full chapter sources + figures (Stages 9.1–9.3).** The chapter
files now contain the complete article text, converted from
`content/book.md` (Stage 9.3): all chapters, the two booktabs tables, the two
equations, the code listings (plain `verbatim`), the rendered Hebrew–English
BiDi passage, and `\cite` commands matching `references.bib`. The two figures
exist as of Stage 9.2 (`figures/architecture.png`,
`figures/generation_time.png`), both deterministically generated by the
scripts under `scripts/`. **No final PDF exists yet and no compile has been
attempted** (TeX Live is not installed). Compilation (Stage 9.4) and PDF
validation (Stage 9.5) are future stages — see `docs/TODO.md` (Stage 9.0
plan) and `docs/DECISIONS.md` (D-030).

## Structure

```
latex_project/
├── main.tex          # entry point: cover page, abstract, ToC, chapters, bibliography
├── preamble.tex      # packages, fonts, Hebrew/English (polyglossia), fancyhdr, biblatex
├── chapters/         # full chapter sources (00_abstract … 08_conclusion), from content/book.md
├── figures/          # architecture.png + generation_time.png (generated, Stage 9.2)
├── scripts/          # generate_architecture.py + generate_graph.py (Stage 9.2)
├── references.bib    # bibliography transcribed from content/book.md only
├── build/            # compile output directory (gitignored; created at build time)
└── content/          # cleaned Markdown source — immutable input, never edited here
```

`content/book.md` is the single source of truth for all article text. The
LaTeX sources are derived from it; the original CrewAI run evidence under
`results/` is never modified.

## Build prerequisites (not yet installed on this machine)

Engine: **XeLaTeX** with polyglossia/bidi for the Hebrew–English section and
DejaVu Sans as the Hebrew font (decision D-030; LuaLaTeX is the fallback).

```bash
sudo apt update
sudo apt install -y texlive-xetex texlive-latex-extra texlive-bibtex-extra biber fonts-dejavu
```

Verify afterwards:

```bash
xelatex --version | head -3
biber --version
fc-match "DejaVu Sans"
```

## Generating the figures (Stage 9.2 — works now)

From the repository root (matplotlib is supplied ephemerally; it is NOT a
project dependency):

```bash
uv run --with matplotlib python latex_project/scripts/generate_architecture.py
uv run --with matplotlib python latex_project/scripts/generate_graph.py
```

The graph values are illustrative inputs, clearly labelled as such — they are
not measured results.

## Build plan (executed from Stage 9.4 on — do not expect a PDF yet)

```bash
# 1. Figures must exist (see above).

# 2. Compile from this directory (multi-pass for ToC + citations):
mkdir -p build
xelatex -interaction=nonstopmode -halt-on-error -output-directory=build main.tex
biber --input-directory build --output-directory build main
xelatex -interaction=nonstopmode -halt-on-error -output-directory=build main.tex
xelatex -interaction=nonstopmode -halt-on-error -output-directory=build main.tex
```

Build artifacts stay in `build/` (gitignored). The final validated PDF is
copied to `results/final_pdf/` only at Stage 9.6.
